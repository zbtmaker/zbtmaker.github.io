Spring Aop动态代理- Prerequisite

发表于 2020-12-31 更新于 2021-02-11

一、动态代理

如何理解代理这个事情，其实就是对原生类进行加强。对于Jdk动态代理，需要三个要素，一个是要被代理的接口，一个是对其增强的InvocationHandler，然后另一个就是Proxy用来生成代理的类。那么我们是否需要回答为什么需要代理这个操作，那我们暂且搁置这个问题。

那么我们看一下Cglib的代理方式，那么就是Enhancer相当于Proxy操作。那么Jdk动态代理的操作是什么呢？就是Method Interceptor，那就齐活了。

现在我们来看一下Spring中是如何处理的呢？Spring为了结合这两者，做了哪些改进呢？第一因为用AopProxy用来抽象Proxy和Enhancer。AopProxy做了什么事情呢？就是提供了一个get Proxy方法，这个方法能够抽象处创建Proxy的接口。

我们都可以看到AopProxy和的继承类。那我们可以看到对于实际的JdkDynamicProxy和CglibAopProxy，里面有个什么呢？在JdkDynamicProxy中有一个AdviseSupport类，这个AdvisedSupport其实就是用来保存一个代理配置的参数和目标类target Class、接口target Interface、以及对应的增强类Advise，其中Advice这个参数其实包含了JointPoint等参数。

那么我们的AdvisedSupport来看一下这一块代码，这里面由AdvisedChainFactory就是为了创建新一个适用于当前Proxy的Advised数组。

第一：为targetClass为了设置对应的advise其实就是对应的advice，这个时候就会把对应的advice全部保存在AdvisedChainFactory。

第二：在调用的时候会触发对应的invoke方法，这个方法就是要首先获取对应List<Advise>，然后在调用的时候。

第三：为什么Advised能够实现责任链设计模式，主要是所有的都继承了JoinPoint接口，这样就能使所Advised能够被调用。

那我们好好捋捋着一块的逻辑，这一块的逻辑主要是为了说明被代理方法之间的调用问题，我们这边因为是调用链方法，所以会出现不能调用的问题。因为是调用链的方式，所以不会调用缓存的方法，但是如果使用context.getBean(A.class)这个时候获取的是被代理的类，此时调用另一个方法，此时调用的方法就会调用invoke方法，然后开始调用责任链的方法。

第二种方式在配置的时候exposeProxy=true。代理类在方法调用之前会把代理类存放在ThreadLocal当中，就是使用AopContext.currentProxy()获取当前，就是使用一个ThreadLocal把当前调用的代理类保存在当前线程里。然后去调用另一个方法的时候调用的就是被代理的方法。

Spring Aop学习笔记三

发表于 2020-10-11 更新于 2021-02-11

前面在学习Aop的时候有些疑问，为什么Aspect已经确定了要织入地方（Pointcut）以及如何织入和执行的地方，为什么还需要一个ProxyFactoryBean来包装相应Aspect和需要被织入的类。原因就是我们需要一个如果一个类想要被织入，那么就需要一个ProxyFactoryBean来生成一个代理类。

一、继承关系

我们可以首先来看一下ProxyFactoryBean的继承关系图。

我们看到这里有ProxyBeanFactory实现了FactoryBean、BeanClassLoaderAware、BeanFactoryAware接口，其中我门知道Aware接口的目的就是实现相应的set方法然后设置ClassLoader和BeanFactory。我们重点需要关注的就是继承的抽象类ProxyCreatorSupport类

1.1 ProxyCreatorSupport

从上面的ProxyCreatorSupport类在初始化的时候会创建一个AopProxyFactory，这个AopProxyFactory主要会根据ProxyFactoryBean的targetClass是否实现了接口而决定使用JDK代理还是Cglib代理方式。我们知道了ProxyFactoryBean的直接父类，那么我们再来看一下AdviseSupport这个父类做了什么事情。

1.2 AdviseSupport

所以这个父类主要是为了主要是为了实现如果一个Proxy被多个Aspect织入的时候，需要此时需要一个List将所有的Advisor按照一定的顺序存储起来。那么在调用的时候是像Tomcat的Filter的调用方式一样吗，对于JdkDynamicProxy方式不是很清楚，但是使用CglibProxy采用的是修改字节码的方式，因此肯定不是。一句话总结：AdvisorSupport主要是多个Advisor织入Proxy时，提供一个执行顺序。

那么我们继续看一下AdvisorSupport继承的父类ProxyConfig和实现的接口Advised。

1.3 ProxyConfig

也就是说这里我们可以在配置的时候就可以用这些参数来确定我们使用什么样的代理方式来确定采用何种代理方式。我们来看一下Advised这个接口做了什么事情，这个接口其实就是定义了ProxyConfig中这几个参数的getter和setter方法。

二、总结

我们来总结一下ProxyFactoryBean能够做什么

  • 设置targetSource，确定了代理的类
  • 在内部创建了AopContextFactory用于创建代理类
  • 将所有的Advisor添加到列表中，并创建新一个AdvisorChainFactory
  • 设置相应的参数proxyTargetClass、optimize、opaque、exposeProxy、frozen等参数

Spring Aop 使用教程

发表于 2020-10-11 更新于 2021-02-11

一、Spring Aop相关概念

1、PointCut解释

Point的作用就是告诉容器在切入哪个类

2、Advice解释

Advice的作用就是告诉容器在哪个类个哪个方法上执行，其实就是解决了When和How的问题，对于When就是告诉容器应该在方法执行之前（BeforeAdvice），方法执行之后（AfterAdvice），在方法抛出异常之后（ThrowingAdvice），以及对应的环绕方法的AroundAdvice，其中AroundAdvice在Spring Aop框架中采用的是MethodInterceptor实现的。

3、Aspect解释

Aspect就是用于整合PointCut和Aspect的，在SpringAOP中并没有对应的Aspect，而是采用类似于Aspect的Advisor接口实现的。一般我们默认采用Default

5、ProxyFactoryBean

我们定义了Advice和PointCut之后，那么如何为代理的类生成相应的代理类呢，这个就是ProxyFactoryBean做的事情。其实ProxyFactoryBean就是一个AOP中Weaver的功能。
