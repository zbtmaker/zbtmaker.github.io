对于二分查找算法，数组一定是全局有序或者是分段有序的。

1、在有序数组中，存在或不存在在重复元素时搜索target元素；

2、在有序数组中，存在重复元素，找到一个最小的索引index使data[index] <= tager；

3、在有序数组中，存在重复元素，找到一个最大的索引index使data[index] > target；

4、在局部有序数组中，不存在重复元素，但是有一个旋转点，旋转点左边有序，旋转点右边有序，旋转点左边的元素大于旋转点右边的元素，找到target元素；

5、在局部有序数组中，存在重复元素，但是有一个旋转点，旋转点左边有序，旋转点右边有序，旋转点左边的元素大于旋转点右边的元素，找到target元素；

LeetCode153

假设按照升序排序的数组在预先未知的某个点上进行了旋转。例如，数组 [0,1,2,4,5,6,7] 可能变为 [4,5,6,7,0,1,2] ，请找出其中最小的元素。

这个题目最直观的方式就是遍历一遍，对于任意的 0 ≤ i < len - 1，其中len是数组的长度。一旦出现nums[i] > nums[i+1]，则返回nums[i+1]。这种暴力遍历的方式，时间复杂度O(N)，空间复杂度O(1)。

对于更快的搜索方法就是二分法了。二分法的精髓就是每一次就是找到寻找区间的中间值，然后确定下一步要搜寻的空间。对于这个题目，给定区间[l, r]，主要遵循下面两个规则：

  • 如果对于nums[mid] < nums[r]，这个时候表示最小值不在应该在[l , mid]；
  • 如果nums[mid] > nums[r]，这个时候因为nums[mid]不是最小值，所以区间应该是[mid + 1, r]；
  • 最终返回nums[l]。

注意，这里我们没有采用nums[mid] 和 nums[left]比较，而是采用了nums[mid] 和 nuns[right]进行比较。主要是当r = l + 1时，mid = l，那么就需要判断nums[mid] = nums[left]的问题，那么这个时候就会加大判断的条件。所以就采用nums[mid] 和 nums[r]进行比较，最终就能减少判断条件。示意图如下
