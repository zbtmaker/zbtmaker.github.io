LeetCode专题-单调队列

发表于 2021-03-20 更新于 2021-05-11

单调队列是一种双端有序队列，在队列中所有的元素都是有序的，队首的元素是最大的，而队尾的元素都是最小的，队列中的元素满足$Q_0>Q_1…Q_n$。单调队列入队的规则就是对于一个索引$0 ≤ i < l < r$,此时$arr[l] > arr[r]$，而此时$l$对$i$的影响要比$r$ 对$l$的影响要大。s

那么什么样的题目应该使用单调队列呢？我想应该是不能得到最直接的答案的。

LC84. 柱状图中最大的矩形

题目描述

给定 n 个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为 1 。求在该柱状图中，能够勾勒出来的矩形的最大面积。

测试用例

暴力解

对于每一个高度柱子，我们从当前柱子的左边和右边找到不低于当前高度的柱子的边界值，我们用 $0 ≤ i <n$，其中 $height[i]$ 表示第 $i$ 根柱子的高度。那么我们要在左区间 $[0, i -1]$ 找到最后一个元素 $k$ 满足条件 $height[k] ≥ height[i]$ ，在右区间$[i+1,n)$中找到最后一个元素 $j$ 满足 $height[j] ≥ height[i]$。按照这种方式能够需要对每一个柱子都需要向两边进行遍历，那么这种方式的时间复杂度为$O(N^2)$，空间复杂度为$O(1)$。

单调栈

前面分析的是对于每一个元素，我们都需要从左右两边找到满足条件的柱子，那么我我们将要求解的问题分成左右两个子问题来求解，那么问题就可以分解成对于柱子 $i$ ，需要在区间$[0, i-1]$找到一个柱子 $0≤ j <i$ 最小能够满足 $height[j] ≥ height[i]$ 。对于每一个元素$0≤j_0≤j_1…≤j_n<i$，如果有$height[j_0] > height[j_1] < height[i]$。那么这个时候就表明对于柱子 $i$ ，其中柱子 $j_1$ 已经不满足条件了。其实这个描述和优先队列的思想很像了，因为我们要从左向右遍历，用一个数据结果存储起来，这样能够利用前面计算好的结果减少后面的计算。

对于一个柱子，如果存在$0≤j_0≤j_1…≤j_M<i$，如果有$height[j_0] > height[j_1] < height[i]$，因为这个时候$height[j_1] > height[i]$，所以对于后面的元素$i < k < n$，其实柱子 $j_1$ 对于 $k$ 已经没有意义了。为什么会得出这样一个结论呢？我们可以做下面的分析

  • 如果 $height[k] > height[i]$，所以柱子 $k$ 的最左边界为 $i+1 $，所以当$0≤j_0≤j_1…≤j_M<i$，如果 $j_M$ 对柱子 $k$ 是一个无意义的。
  • 如果 $height[k] < height[i]$，此时柱子 $k$ 的最左边界就是一直将栈中的元素移除堆栈，直到碰到一个柱子的 $height[j_M]< height[k]$ 。那么此时 $k$ 的左边最大面积为 $(k - j_M)*height[k]$。

还存在一个问题，就是 $0≤j_0≤j_1…≤j_n<i$，如果碰到 $height[i] == height[j_n]$ ，那么应该怎么做呢？也是要出栈的，因为通过上面的分析我们要找到的是第一个 $height[j_M]< height[k]$，所以如果高度相等则从栈中弹出。

LC239. Sliding Window Maximum

这个题目是一道披着滑动窗口的外衣，实际上是一个优先队列的题目。这里我们采用双端队列构造一个单调队列存储窗口内的数据，入队规则如下：

  • 添加的元素都从队尾进入，而如果双端队列尾部元素小于当前元素，则将队尾元素移出队列，直到$queue.size == 0\ || \ queue.last \geq e$；
  • 每次我们都会取队首的元素作为窗口的最大值放入到结果数组中。
  • 当队列满的情况下，此时需要判断我们的最大值是否是窗口最右边值，如果当前窗口的最左边值等于最大值，那么在下一个窗口会被移出的

为什么我们需要维持这样一个队列呢？是因为我们需要的是找到窗口的最大值，后进入的队列为什么要把队尾的元素先出队，假设$i < j$，且$nums[i] < nums[j]$，那么随着队列向右滑动，这个$nums[i]$就用不上了，因为$i <j$，所以$nums[i]$肯定会被先移出队列。所以我们将这个操作前置，而且能够减少队列的复杂度。

另一个规则，如果$i < j$，但是$nums[i] > nums[j]$，此时为什么$nums[j]$要入队呢？这是因为很有可能$nums[i]$很可能在下一个滑动就被移出队列，而$nums[j]$很可能就是下一个最大值。

LC739. 每日温度

题目描述： 每日温度

这个题目要做的就是对于每一个$i$ ，其中$[0,len]$，我们要在$[i+ 1,len]$找到一个第一个温度使得$temperature[i] < temperature[j]$。但是其实我们可以从后往前看，对于最后一个元素肯定是不会有温度比其高的。

  • 如果我们用一个单调队列维护$(temperature,index)$对来记录当前值所有升高的$temperature$和$index$，一旦有一个新的元素要添加到单调队列时，需要将队尾所有小于等于当前$temperature$的温度从队列中移除。
  • 当$l < r$，但是$temperature[l] > temperature[r]$，那么对于任意一个索引$0 ≤ i < l < r$，此时$temperature[l]$对于$i$来说是最近的一个会升高的温度，因此我们就不需要存储$temperature[r]$(于是就需要移出队列)。

LC862. Shortest Subarray with Sum at Least K

题目描述：和至少为 K 的最短子数组

我们第一步是先求出数组的前缀和，然后采用暴力的方式解决这个问题，从暴力解中，时间复杂度$O(N^2)$，空间复杂度$O(N)$。首先我们来看一下暴力解，这种方式超时了，还有10多个case无法通过。

那么我们想到区间和，那么有一种很直观的方式就是采用线断树来做，这个我们不在这里尝试，我们尝试使用单调队列的方式来实现。

我们用 $preSum[i]$表示区间$[0,i]$的和，那么对于区间和$(i,j] = preSum[j] - preSum[i]$。如果我们想要在一个区间找到满足题目条件的区间和 $preSum[j] - preSum[i] ≥ k$ ，那么也就是说 $preSum[j] ≥ preSum[i] + k$，对于索引 $j$ 来说，其实就是找到第一个满足$preSum[j] ≥ preSum[i] + k$的索引。

  • $0 < l < i$，如果 $preSum[i] < preSum[l]$，此时索引l对于索引 $j$ 就没用了；因此可以从单调队列中出队了；
  • $0< l < i$，如果 $preSum[i] > preSum[l]$ ，如果那么此时 $preSum[m]$这个值还有没有用呢？其实，这个时候我们需要的就是从队列将队首的元素出队，因为这个时候队首可能还会有比$preSum[i]$大的前缀和。

参考链接：https://github.com/Shellbye/Shellbye.github.io/issues/41

所以我们从这个题目就确定了入队的操作。确定入队操纵之后，需要计算队列中存储的元素，我们存储索引值，而不是直接存储前缀和.

LC901. 股票价格跨度

题目描述：股票价格跨度

对于$\forall\ i$ 有$0 < i < len$，我们要在区间$[0,i]$内找到连续的元素使得$stock[i] ≥ stock[j],\forall 0 \leq j \leq i$。

  • 为什么会想到用单调队列呢？这是因为我们在采用暴力过程中我们从前往后遍历，其实每一次就只需要将大于当前股票的第一个位置的股票位置找到即可。

  • 一旦有$0 \leq l< r <i $，此时如果有$stock[r] < stock[l]$，那么我们对于后面出现的股票价格$stock[i]$而言，此时$stock[r]$比$stock[l]$更有价值。而队列中不应该再存储比$stock[r]$小的任何股票。

  • 我们应该用一个$Pair(price,count)$，其中$price$表示当前的股票价格，而$count$则表示连续小于或等于当前$price$的数量。
