LeetCode专题-单调队列

发表于 2021-03-20 更新于 2021-03-28

单调队列是一种双端有序队列，在队列中所有的元素都是有序的，队首的元素是最大的，而队尾的元素都是最小的，队列中的元素满足$Q_0>Q_1…Q_n$。单调队列入队的规则就是对于一个索引$0 ≤ i < l < r$,此时$arr[l] > arr[r]$，而此时$l$对$i$的影响要比$r$ 对$l$的影响要大。

那么什么样的题目应该使用单调队列呢？我想应该是不能得到最直接的答案的

LC739. 每日温度

题目描述： 每日温度

这个题目要做的就是对于每一个$i$ ，其中$[0,len]$，我们要在$[i+ 1,len]$找到一个第一个温度使得$temperature[i] < temperature[j]$。但是其实我们可以从后往前看，对于最后一个元素肯定是不会有温度比其高的。如果我们用一个单调队列维护$(temperature,index)$对来记录当前值所有升高的$temperature$和$index$，一旦有一个新的元素要添加到单调队列时，需要将队尾所有小于等于当前$temperature$的温度从队列中移除。比如$l < r$，但是$temperature[l] > temperature[r]$，那么对于任意一个索引$0 ≤ i < l < r$，此时$temperature[l]$对于$i$来说是最近的一个会升高的温度。

LC862. Shortest Subarray with Sum at Least K

题目描述：和至少为 K 的最短子数组

我们第一步是先求出数组的前缀和，然后采用暴力的方式解决这个问题，从暴力解中，时间复杂度$O(N^2)$，空间复杂度$O(N)$。首先我们来看一下暴力解，这种方式超时了，还有10多个case无法通过。

那么我们想到区间和，那么有一种很直观的方式就是采用线断树来做，这个我们不在这里尝试，我们尝试使用单调队列的方式来实现。

我们用 $preSum[i]$表示区间$[0,i]$的和，那么对于区间和$(i,j] = preSum[j] - preSum[i]$。如果我们想要在一个区间找到满足题目条件的区间和 $preSum[j] - preSum[i] ≥ k$ ，那么也就是说 $preSum[j] ≥ preSum[i] + k$，对于索引 $j$ 来说，其实就是找到第一个满足$preSum[j] ≥ preSum[i] + k$的索引。

  • $0 < l < i$，如果 $preSum[i] < preSum[l]$，此时索引l对于索引 $j$ 就没用了；因此可以从单调队列中出队了；
  • $0< l < i$，如果 $preSum[i] > preSum[l]$ ，如果那么此时 $preSum[m]$这个值还有没有用呢？其实，这个时候我们需要的就是从队列将队首的元素出队，因为这个时候队首可能还会有比$preSum[i]$大的前缀和。

参考链接：https://github.com/Shellbye/Shellbye.github.io/issues/41

所以我们从这个题目就确定了入队的操作。确定入队操纵之后，需要计算队列中存储的元素，我们存储索引值，而不是直接存储前缀和.

LC901. 股票价格跨度

题目描述：股票价格跨度

对于$\forall\ i$ 有$0 < i < len$，我们要在区间$[0,i]$内找到连续的元素使得$stock[i] ≥ stock[j],\forall 0 \leq j \leq i$。

  • 为什么会想到用单调队列呢？这是因为我们在采用暴力过程中我们从前往后遍历，其实每一次就只需要将大于当前股票的第一个位置的股票位置找到即可。

  • 一旦有$0 \leq l< r <i $，此时如果有$stock[r] < stock[l]$，那么我们对于后面出现的股票价格$stock[i]$而言，此时$stock[r]$比$stock[l]$更有价值。而队列中不应该再存储比$stock[r]$小的任何股票。

  • 我们应该用一个$Pair(price,count)$，其中$price$表示当前的股票价格，而$count$则表示连续小于或等于当前$price$的数量。
